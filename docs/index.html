Overview

What is this application?

RAG Knowledge Base is an AI-powered assistant that reads your documents and answers questions about them. Instead of searching through files manually, you simply ask a question and the system finds the most relevant passages, then uses an AI model to compose a clear answer — with references back to the original source.

📄

Upload Documents

Drag and drop PDFs, Markdown files, or plain text. The system accepts the formats you already use.

💬

Ask Questions

Type a question in plain English. No special syntax, no keywords — just ask like you'd ask a colleague.

✓️

Get Cited Answers

Every answer includes numbered citations linking back to the exact document passages it drew from.

The RAG Pipeline

How does it work?

The system uses a technique called Retrieval Augmented Generation (RAG). Here's the journey your document takes, and what happens when you ask a question.

Part 1 — When you upload a document

📄 Upload File PDF, Markdown, Text
➔
🔎 Extract Text Read every page
➔
✄️ Split into Chunks ~512 characters each
➔
🧮 Create Embeddings Numbers that capture meaning
➔
🗃️ Store in Database Ready for search

💡 What are embeddings?

Think of embeddings as a way to convert words into coordinates on a map. Sentences with similar meanings end up close together on that map. When you ask a question, the system converts it into the same type of coordinates and finds the closest document passages — even if you used completely different words.

Part 2 — When you ask a question

1

Your question gets embedded

The same embedding model converts your question into coordinates, so it can be compared to the stored document passages.

2

Nearest passages are found

A lightning-fast vector search scans every stored chunk and returns the 5 most relevant passages in under a millisecond.

3

A smart prompt is assembled

The system packages your question, the retrieved passages, and the conversation history into a structured prompt for the AI model.

4

The AI composes an answer

The language model reads the context and generates a coherent answer, streaming it word by word to your screen. Each claim is tied to a numbered citation so you can verify it.

Features

Everything you need

Built for real-world use, not just demos.

📁 Workspaces

Organize documents into separate workspaces — one per project, client, or topic. Each workspace has its own documents and conversations.

⚡ Streaming Responses

Answers appear word by word in real time, just like a chat app. No waiting for the full response to generate before you see anything.

🔗 Source Citations

Every answer includes clickable citation markers [1], [2], etc. Expand them to see the exact source passage, filename, and page number.

🔄 Document Reindexing

Changed your chunking strategy or embedding model? One click to reindex any document without re-uploading it.

🛡️ Rate Limiting

Built-in protection against excessive use with a sliding-window rate limiter backed by Redis.

🎬 Swap AI Models Freely

Switch between a local Ollama model and OpenAI by changing two environment variables. No code changes needed.

Architecture

What's running under the hood

Six services work together, all orchestrated by Docker Compose and started with a single command.

🌐 Frontend

Next.js app with TypeScript and Tailwind. The user interface for everything.

localhost:3000

⚡ Backend API

Python FastAPI server handling all requests, file uploads, and SSE streaming.

localhost:8000

⚙️ Worker

Celery background worker that processes documents, chunks text, and generates embeddings.

background process

🗃️ PostgreSQL + pgvector

Stores all data — workspaces, documents, chunks with vector embeddings, conversations, and messages.

localhost:5432

🔥 Redis

Task queue broker for background jobs, caching layer, and rate limiting backend.

localhost:6379

🤖 Ollama

Local large language model server. Runs llama3.2 (or any model you choose) entirely on your machine.

localhost:11434

Getting Started

Up and running in 3 minutes

All you need is Docker installed on your machine. Everything else runs inside containers.

# 1. Navigate into the project
$ cd rag-knowledge-base

# 2. Copy the example environment file
$ cp .env.example .env

# 3. Start all 6 services (Postgres, Redis, Backend, Worker, Frontend, Ollama)
$ make up

# 4. Download the AI model into Ollama (first time only, ~2 GB)
$ make pull-model

# ✨ That's it! Open http://localhost:3000

Switching AI providers

🤖 Ollama (default, fully local)

Runs entirely on your machine. No API keys, no internet needed after model download.

LLM_MODEL=ollama/llama3.2
OLLAMA_API_BASE=http://ollama:11434

☁️ OpenAI (cloud)

Uses OpenAI's API for higher quality responses. Requires an API key.

LLM_MODEL=openai/gpt-4o-mini
OPENAI_API_KEY=sk-...
User Journey

Your typical workflow

Here's what a normal session looks like, step by step.

1

Create a workspace

Name it after your project, client, or research topic. Workspaces keep your documents and conversations organized.

2

Upload your documents

Drag and drop PDFs, Markdown, or text files onto the upload area. Each file is processed automatically in the background — the status badge shows the progress.

3

Wait for the green badge

Once a document's badge turns ready, its contents have been parsed, chunked, and embedded. It's ready for questions.

4

Start a conversation

Click New Chat and type your question. The answer streams in real time with numbered citations.

5

Verify with citations

Click any [1], [2] marker to expand the source passage. See exactly which document and page the information came from.
